<objective>
Create the Phase 8 backend foundation for safe sharing by introducing a dedicated share-record model and authenticated owner APIs to manage public links for saved tasks.

Purpose: Public access should not be inferred from the existing task shell metadata. Phase 8 needs a separate share boundary that can be created, rotated, and revoked without changing the owner task-detail contract or the Phase 7 present-mode save boundary.
Output: A migrated share table plus owner-authenticated APIs that can create, list, regenerate, and revoke one explicit public view link and one explicit public present link per saved task.
</objective>
